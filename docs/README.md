# Docs Guide

This folder contains both active project documentation and historical/reference material.

## Protected Docs

Do not delete, archive, rename, or replace these files during cleanup unless the user explicitly asks:

- `docs/dev-log.md`
- `docs/implementation-plan.md`
- `docs/performance-guardrails.md`
- `docs/mobile-execution-checklist.md`
- `docs/mobile-smoke-test-matrix.md`
- `docs/mobile-release-checklist.md`
- `docs/api-contract.md`
- `docs/StudyHub.postman_collection.json`

These files are important because they act as:

- current project source of truth
- handoff memory between sessions
- lesson-defense/demo artifacts
- active verification checklists

## Reference Docs

Usually keep these too, but ask before deleting if a cleanup task targets them:

- `docs/assignment.md`
- `docs/social-features-plan.md`
- `docs/audit-fix-plan.md`
- `docs/mobile-phone-testing-handoff.md`
- lesson/reference PDFs placed in `docs/`

## Cleanup Guidance

Good cleanup candidates usually need explicit review first:

- old `.bak` files
- superseded repaired/corrupted backup docs
- obsolete scratch notes
- archived legacy notes no longer needed for defense or implementation

If a future cleanup pass is unsure whether a doc is temporary, default to keeping it.
